# motion-tuner

マイクロアニメーションのチューニング＆ストックアプリ。
Claudeとの会話で作ったキーフレームCSSを貼り付けるだけで、パラメータ調整・プレビュー・ストック・クライアント共有・実装用エクスポートまで行える。

- ビルド不要（素のHTML/CSS/JSのみ。CDN経由のWebフォントのみ使用）
- サーバー・DB不要（永続化はブラウザのlocalStorageのみ）

## 使い方

### 起動

`index.html` をブラウザで開くだけ。初回起動時はプリセット6種（フェードイン／エラスティック登場／パルス／シェイク／モーダル「ぼわん」／スタッガーフェードイン）と、複合アニメーション1種（但馬 THINK into カルーセル）がライブラリに読み込まれる。

ヘッダーのトグルでライト/ダークテーマを切り替えられる（デフォルトはライト。選択はlocalStorageに保存され、次回以降も維持される）。

ライブラリの各カードにはミニプレビューがあり、カードにマウスホバーすると再生される（無限ループ系はホバー中のみ繰り返し）。タッチデバイスやキーボード操作では、カード内の「▶」ボタンで再生できる。

### 新規追加（貼り付け）

「新規追加」画面のテキストエリアに以下のどちらかを貼り付けて「読み込んでチューニングへ」を押す。

- **パターンA：JSONメタ情報つき** … `/* { "name": ..., "params": {...} } */` コメント＋`@keyframes`。全フィールドが自動入力される
- **パターンB：素のCSS** … `@keyframes` ブロックのみ（または中身だけ）。`var(--xxx)` を自動検出してスライダー化される（min/max/defaultは単位から仮決め）

「Claude用の指示文をコピー」ボタンで、パターンA形式で書いてもらうための依頼文をコピーできる。

「数式ジェネレーター：減衰振動スウィングを追加」ボタンを押すと、数式（減衰×正弦波）から`@keyframes`を自動生成するアニメーションが追加される。振れ幅（初動／戻り）・減衰速度・振動周波数のスライダーを動かすたびにkeyframesが再計算されるので、「電車が止まる感じの揺れ」のような手書き不可能なカーブもチューニングできる（keyframes欄は自動生成のため編集不可）。

### 複合アニメーション（親）

カルーセルのように複数要素・状態管理・JS制御を含む動きは、1つの`@keyframes`に分解できないため「複合アニメーション」として自己完結HTML一式のまま保管する。

- **追加** … 「新規追加」→「複合アニメーション」タブにClaudeが出力した1ファイルHTMLを貼り付けて保存。先頭に `<!-- { "name": ..., "description": ... } -->` 形式のメタコメントがあれば名前・説明が自動入力される。「Claude用の指示文をコピー（複合アニメーション）」でこの形式で書いてもらう依頼文をコピーできる
- **一覧での親子表示** … ライブラリでは複合アニメーションが「複合」バッジ付きの親カードとして表示され、関連づけたアニメーションが接続線つきでその下にぶら下がる。どれが親でどれが関連ファイルかが一目で分かる
- **関連づけ** … 親カードの「関連アニメを追加」から新規登録するか、各アニメーションのチューニング画面の「親の複合アニメーション」セレクトでいつでも付け替えられる
- **プレビュー** … 「プレビュー」でモーダル内のiframeに完成形を表示。「新しいタブで開く」で全画面確認もできる
- **完了ステータス** … 親カードの「完了にする」を押すと、一番下の折りたたみセクション「完了した複合アニメーション」に自動で移動する（「進行中に戻す」で復帰）
- **並び替え** … 親カード・単体アニメーション・関連アニメーションのカードにある「↑↓」ボタンで、それぞれのグループ内の表示順を入れ替えられる
- **Cursor連携** … 「Cursorブリーフをコピー」で、説明＋関連アニメーションの調整値＋リファレンスHTML一式を含む実装依頼文がコピーされ、Cursorのチャットにそのまま貼れる
- **共有** … 複合アニメーションはURL共有には大きすぎるため、「ダウンロード」でHTMLを書き出し、リポジトリの `prototypes/` などに置いてデプロイして共有する

### チューニング

- duration / delay / カスタムパラメータはスライダーと数値入力で調整
- イージングはプリセット（linear〜back風・バウンス風）またはカスタムcubic-bezier。カーブはSVGグラフにリアルタイム描画される
- ループ（無限 or 回数指定）、スタッガー（要素数×間隔）はチェックボックスでON/OFF
- **トリガー種別**（ページロード時／クリック時／ホバー時／スクロール到達時）を選ぶと、プレビューの発火方法がそのトリガーに合わせて変わる
  - クリック時 → プレビューをクリックで発火／ホバー時 → ホバーで発火（無限ループ系はホバー解除で停止）
  - スクロール到達時 → チューニング中は調整しやすいよう要素を常時表示（変更は即再生）。「スクロール発火を試す」ボタンでスクロール領域＋IntersectionObserverによる実際の発火をシミュレーションできる
  - 「リプレイ」ボタンはトリガー種別に関係なくいつでも手動再生できる
  - 自由記述の「トリガー詳細」は実装者への申し送りメモとして別に残せる
- プレビュー・イージングカーブの列は画面に追従（sticky）するので、下の方の項目を調整しながらでも動きを確認できる
- 変更は自動保存される

### 共有

ライブラリ各カードの「共有リンクを作成」で、データをBase64エンコードした `share.html?data=...` のURLがコピーされる。開いた相手にはそのアニメーション1つだけが読み取り専用で表示される。URLが5000文字を超える場合は警告が出る。

共有ページにはプレビューに加えて「仕様」（トリガー・対象セレクタ・duration・delay・easing・カスタムパラメータ・備考）と、折りたたみの「使用しているCSS」（実値解決済みのCSSドキュメント、コピー可）が表示されるので、受け取った側はそのまま実装に使える。

### エクスポート（チューニング画面から）

1. **Obsidian仕様書Markdown** … トリガー種別・詳細／duration／easing／対象要素／パラメータ一覧／発火用JS／備考
2. **CSSドキュメント** … `:root` のカスタムプロパティ＋`@keyframes`＋実装例クラス＋発火用JSスニペット（コピペで使える形）
3. **Cursor実装ブリーフ** … 実値解決済みのCSSとトリガー種別に応じた発火用JSを含む、Cursorのチャットにそのまま貼れる依頼文

## デプロイ手順（GitHub Pages ＋ 独自サブドメイン）

このリポジトリには `motion.tsubame-tora.jp` を1行だけ書いた `CNAME` ファイルを同梱済み。以下の手順でデプロイする。

1. GitHubに `motion-tuner` という新規リポジトリを作成し、このディレクトリの中身をpushする

   ```bash
   cd motion-tuner
   git init
   git add .
   git commit -m "initial commit"
   git remote add origin git@github.com:<ユーザー名>/motion-tuner.git
   git push -u origin main
   ```

2. リポジトリの **Settings → Pages** で GitHub Pages を有効化する
   - Source: 「Deploy from a branch」
   - Branch: `main` / `/ (root)` を選択して Save
   - Custom domain 欄に `motion.tsubame-tora.jp` が表示されていることを確認する（`CNAME` ファイルから自動反映される）

3. ドメイン管理画面（お名前.com等）で `motion` サブドメインのCNAMEレコードをGitHub Pages側に向ける

   | ホスト名 | 種別 | 値 |
   | --- | --- | --- |
   | `motion` | CNAME | `<ユーザー名>.github.io.` |

4. 反映まで数分〜数時間待ち、`https://motion.tsubame-tora.jp` にアクセスして動作確認する
   - Settings → Pages の「Enforce HTTPS」にチェックを入れる（証明書発行後に有効化できる）

## ファイル構成

```
motion-tuner/
├── index.html      … ライブラリ一覧＋新規追加＋チューニング画面（自分用）
├── share.html      … 個別共有ビュー（URLパラメータから復元、読み取り専用）
├── style.css
├── app.js          … ライブラリ管理・貼り付けパーサー・チューニングロジック
├── animations.js   … デフォルトで最初から入っている6種類のプリセットデータ
├── prototypes.js   … デフォルトの複合アニメーション（但馬カルーセルのHTML一式を内包）
└── CNAME           … 中身は motion.tsubame-tora.jp の1行のみ
```

## 補足

- localStorageのキーは `motion-tuner:library`（単体アニメーション）、`motion-tuner:prototypes`（複合アニメーション）、`motion-tuner:theme`（テーマ選択）。全データを消したい場合はブラウザの開発者ツールから削除する
- 6種類のプリセットはすべて共通のデータ構造・共通のレンダリングエンジンで処理されており、種類ごとの特別扱いはない。スタッガーは「種類」ではなくどのアニメーションでもONにできる「モード」
